# Phase 1 — Requirements Engineering

Mục tiêu: Hệ thống cần làm gì?

Phase này bám ISO/IEC/IEEE 29148.

## Artifact

```text

1.1 Stakeholder Needs
1.2 Business Requirements
1.3 User Requirements
1.4 System Requirements
1.5 Functional Requirements
1.6 Non-functional Requirements
1.7 Use Case Specification
1.8 Acceptance Criteria
1.9 Requirements Traceability Matrix

```

### Example

Business Requirement:
- Khoa cần một hệ thống hỗ trợ ôn luyện Olympic tập trung.

User Requirement:
- Sinh viên cần làm đề và biết kết quả ngay.

System Requirement:
- Hệ thống phải cho phép sinh viên nộp bài và tự động tính điểm.

Functional Requirement:
- Khi student submit exam attempt, system calculates score based on correct answers.

Non-functional Requirement:
- Kết quả phải trả về trong vòng 2 giây với đề <= 100 câu.

## Exit criteria
- Có danh sách requirement có ID.
- Requirement không mơ hồ.
- Có acceptance criteria.
- Có phân loại priority: Must / Should / Could / Won’t.
- Có trace từ business goal -> user need -> system requirement.
